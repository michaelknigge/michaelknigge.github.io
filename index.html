My Expertise

In my more than 20-year career as a software developer, I have been developing software on virtually any platform that is relevant to the business world. I've developed various application on OS/2 and Windows, a LU 6.2 application on Novell Netware, libraries and batch applications on Linux, SunOS, AIX, HP-UX and some batch and dialogue applications on the mainframe operating systems z/OS and BS2000. Currently, I mainly work on the following platforms

Windows

Windows is my daily platform I work on. Currently I use Eclipse to do my daily work as a Java developer. But in the past I also coded some more native stuff (native Win32 applications, managed applications with C#, Windows services and also some port monitors or filter drivers).

Linux

I also do some of my daily work on Linux (and sometimes AIX and SunOS, too). In the past I've been also working on those platforms and developed software in plain C. But currently those platforms are just platforms I do some unit and integration tests, system managament and so on.

z/OS

Oh yeah, I love z/OS. Ok, mostly. Sometimes it really hurts. But I love the platform for its availability and stability. And it is really fun and thrilling to combine the old classic mainframe world (MVS environment) to the new open world (Unix System Services - the environment Java runs in).

Featured Projects

pclbox

pclbox is a Java library that is able to completely parse PCL printer data streams (up to version 5). It can also handle embedded HP-GL/2 as well a PJL.

Check it out
pcldumper

pcldumper can produce a human readable dump of a PCL priter data stream. It uses pclbox to do its work.

Check it out
afpbox

afpbox is a planned project. Parsing AFP printer data streams is part of my daily job. I came across some java libraries that can handle and parse AFP, but I'm not satisfied with them. They are sometimes complicates, sometimes they have huge dependencies and sometimes they are just buggy crap (sorry). I'm currently thinking about coding my own library. But I guess this will take some time (I'm nut just a coder, I'm a father too).

Check it out
ipdsbox

I'm some kind of a printer data stream and protocol junkie. And I always want to know what happens behind the scenes. I've started to work on a Java library that can parse IPDS data streams. But the project is currently not active and far away from being complete. I currently thinking about developing afpbox and ipdsbox in parallel because they have many commonalities. But as said already, I'm a father too (and husband) and my family is much more valuable to me as a Java library. So things will need some time, maybe one or two years.
